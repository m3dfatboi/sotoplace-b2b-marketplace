# DESIGN BRIEF: B2B Marketplace + ERP/CRM Platform

**Дата создания:** 2026-03-03
**Статус:** Активный дизайн-бриф для всех работ по UI/UX

---

## КОНТЕКСТ ПРОДУКТА

**Платформа:** B2B-маркетплейс + ERP/CRM полного цикла для производственных/торговых компаний.

**Главная особенность:** Мультитенантность + гибридные роли; один контрагент может быть покупателем/продавцом/производителем/поставщиком/исполнителем.

**UI динамически адаптируется** под роль сотрудника внутри компании-контрагента.

---

## РОЛИ И РЕЖИМЫ (обязательные UX-правила)

### 1) Менеджер (продажи/операции)
- **На главной:** аналитика (воронка, нагрузка, лидеры), активные счета/сделки со статусами оплат/отгрузок/производства
- **Внутри сделки:** комплектация, поиск поставщиков/логистика, суб-сделки, закупочные цены (внутренний вид)

### 2) Конструктор (инжиниринг)
- Задачи на проектирование, загрузка чертежей (CAD/PDF) в карточку счета
- Специализированный чат по конкретному чертежу/версии, комментирование версий
- Кнопка "Согласовать/Approve", после чего чертеж уходит в производство

### 3) Клиент/Заказчик
- Видит маркетплейс-каталог, корзину, статусы своих заказов
- **Не видит** внутренних процессов сборки и прямых поставщиков продавца (конфиденциальность цепочки поставок)

### 4) Администратор компании
- Профиль организации, реквизиты/счета, управление сотрудниками и правами
- Гостевые входы, настройки публичного каталога

## МОДУЛИ

### A) Каталог и Маркетплейс
- Публикация товаров/услуг/мощностей
- Карточки: артикулы, цены, характеристики, комплектации, фото
- QR-коды для быстрого доступа к позиции из оффлайна
- Корзина, оформление сборных заказов

### B) Управление заказами и счетами (Сделки)
- Сложный пайплайн статусов от черновика до отгрузки
- Позиции могут разбиваться на детали ("Стол" = "Каркас" + "Столешница")
- Для каждой детали поиск отдельного подрядчика
- Визуализация готовности по операциям (ТР, ОК, УП, СК, СБ) на уровне каждой позиции/детали
- **Мультипользовательская работа:** показывать, кто ещё смотрит сделку сейчас

### C) Инжиниринг и согласование
- Загрузка CAD/PDF, версионирование, предпросмотр
- Мессенджер, привязанный к чертежу/заказу, комментарии по версиям
- Электронное согласование → дальнейшие этапы производства/подрядчиков

### D) База контрагентов и "биржа труда"
- Реестр компаний с тегами по специализации
- **Безопасные сделки:** сначала анонимно; после утверждения — открытие контактов
- Поиск временных исполнителей на проектные работы

## БАЗОВЫЙ СЦЕНАРИЙ (Customer Journey)

1. **Поиск товара** в каталоге
2. **Инициация:** стандарт → корзина; кастом → чат с продавцом
3. **Проектирование:** конструктор делает чертеж → загрузка → клиент "Согласовать"
4. **Комплектация:** менеджер разбивает на детали → ищет подрядчиков в базе → появляются закупочные цены (внутренний слой)
5. **Производство/логистика:** субподрядчики выполняют; клиент видит только общий прогресс без раскрытия цепочки
6. **Завершение:** доставка/закрытие заказа → обновление финансовой аналитики

## ВИЗУАЛЬНЫЙ СТИЛЬ И КАЧЕСТВО

### Стиль
- Современный B2B SaaS 2025–2026: чисто, уверенно, "дорого", без перегруза
- Mobile-first, адаптивность: **360/390, 768, 1024, 1440**
- **8pt spacing system**, консистентные радиусы/бордеры/тени

### Accessibility (A11y)
- Контраст, фокус-стейты, клавиатура
- Понятные ошибки/валидация

### Данные
- Приоритет таблиц, фильтров, статусов
- Сохранённые представления, массовые действия

### Микроинтеракции
- hover/active/disabled/loading
- Skeletons, empty/error states
- Анимации **150–220ms**

## ОСНОВНЫЕ СТРАНИЦЫ MVP

### 1) App Shell / Навигация (общая рамка)
- Переключение тенанта (компания)
- Переключение роли/режима (если доступно)
- Поиск по системе
- Sidebar/topbar, breadcrumbs, уведомления, "быстрые действия"
- **Ясное разделение** "внешнего клиентского" и "внутреннего операционного" слоёв

### 2) Dashboard (разный по ролям)

**Менеджер:**
- KPI, воронка, активные сделки
- Алерты SLA, задачи/нагрузка

**Конструктор:**
- Очередь задач
- Последние комментарии по чертежам
- Запросы на согласование, статусные блокеры

**Клиент:**
- Статус активных заказов (упрощённо)
- Уведомления по согласованию
- Рекомендации/повтор заказа

**Админ:**
- Здоровье организации (профиль заполнен/нет)
- Пользователи/роли
- Настройки каталога

### 3) Каталог / Маркетплейс
- Поиск, фильтры, сортировка
- Сохранённые фильтры/вьюхи
- Карточка товара/услуги/мощности: комплектации, фото, характеристики
- CTA (в корзину / запросить кастом / написать)
- **QR entry:** экран "товар по QR" (быстрый просмотр + действие)

### 4) Корзина и оформление
- Сборный заказ
- Проверка доступности/сроков
- Варианты доставки/самовывоза
- Комментарии

### 5) Сделки / Заказы (список)
- Таблица: статусы, SLA, суммы, контрагенты, этапы
- Фильтры, сохранённые виды, массовые действия

### 6) Сделка / Заказ (detail)
- Таймлайн статусов
- Позиции/детали
- Операции ТР/ОК/УП/СК/СБ (визуально ясно)

**Внутренний вид (менеджер):**
- Суб-сделки, подрядчики, закупочные цены, логистика

**Внешний вид (клиент):**
- Общий прогресс без раскрытия подрядчиков/закупочных цен

**Присутствие коллег:**
- Аватары/индикатор "сейчас в сделке"
- Аккуратные подсказки

### 7) Инжиниринг: чертежи и согласование
- Загрузка/версии, предпросмотр
- Комментарии к версии
- Чат по чертежу
- Кнопка "Согласовать" + юридически/UX корректное подтверждение

### 8) База контрагентов / Поиск подрядчиков
- Теги/специализации, фильтры
- Карточки компаний
- Режим "анонимно" → "контакты раскрыты" по событию/статусу сделки

### 9) Настройки компании и права
- Профиль, реквизиты, счета
- Пользователи/роли/права
- Гостевые входы
- Настройки публичного каталога

## ТРЕБОВАНИЯ К РЕАЛИЗАЦИИ

### Адаптивность
- **360px** - минимальная ширина (mobile)
- **390px** - стандартный mobile
- **768px** - tablet
- **1024px** - desktop small
- **1440px** - desktop large

### Spacing System
- **8pt grid** - все отступы кратны 8px
- Базовые значения: 8, 16, 24, 32, 40, 48, 64, 80, 96

### Typography
- Fluid typography с адаптивными размерами
- Чёткая иерархия H1/H2/H3/body/small
- Monospace для кодов/артикулов

### Colors
- Нейтральная шкала (50-950)
- Primary/secondary
- Success/warning/danger/info
- Background/surface/border/text tokens

### Animations
- **150-220ms** для микроинтеракций
- Easing: cubic-bezier(0.4, 0, 0.2, 1)
