## Ключевые концепции системы

### 1. Мультитенантность
- Каждая компания (company) - отдельный tenant
- Пользователи могут быть в нескольких компаниях с разными ролями
- Изоляция данных через `company_id` в большинстве таблиц

### 2. Роли и права доступа
```
- Admin: Управление компанией, настройки, права доступа
- Manager: Управление сделками, аналитика, поиск поставщиков
- Constructor: Работа с чертежами, техническое согласование
- Client: Просмотр каталога, создание заказов, отслеживание статусов
```

### 3. Жизненный цикл заказа
```
draft → negotiation → approved → in_production → assembly →
quality_check → shipping → delivered → completed
```

### 4. Вложенная структура
- Заказы могут иметь субзаказы (parent_order_id)
- Позиции заказа могут разбиваться на детали (parent_item_id)
- Каждая деталь может иметь своего субподрядчика

### 5. Конфиденциальность
- Клиент НЕ видит субподрядчиков продавца
- Контакты контрагентов открываются только после согласования
- Чертежи доступны только участникам сделки

## Основные таблицы БД

### Пользователи и компании
- `users` - пользователи системы
- `companies` - компании (tenants)
- `company_members` - связь пользователей с компаниями + роли

### Каталог и товары
- `products` - товары/услуги компаний
- `product_variants` - варианты комплектации
- `product_tags` - теги для поиска

### Заказы и сделки
- `orders` - основная таблица заказов
- `order_items` - позиции в заказе
- `order_item_suppliers` - связь позиций с субподрядчиками

### Коммуникация
- `chats` - чаты (привязаны к заказу/чертежу)
- `chat_participants` - участники чатов
- `messages` - сообщения
- `blueprints` - чертежи
- `blueprint_versions` - версии чертежей
- `blueprint_approvals` - согласования

### Контрагенты
- `company_relationships` - связи между компаниями
- `contractor_requests` - заявки на поиск исполнителей
- `contractor_responses` - отклики на заявки

### Аналитика и система
- `company_analytics` - агрегированная аналитика
- `order_analytics` - метрики по заказам
- `user_performance` - метрики пользователей
- `notifications` - уведомления
- `audit_logs` - журнал действий
- `entity_history` - версионирование объектов

## Технические детали

### Backend Stack
- FastAPI (async)
- PostgreSQL 15+
- SQLAlchemy 2.0 (async)
- Alembic (миграции)
- Pydantic v2 (валидация)
- Redis (кэш, очереди)
- Celery (фоновые задачи)

### Паттерны проектирования
- Repository pattern для работы с БД
- Service layer для бизнес-логики
- Dependency Injection через FastAPI
- JWT для аутентификации
- RBAC (Role-Based Access Control)

### Безопасность
- Хеширование паролей (bcrypt)
- JWT токены с refresh
- Rate limiting
- CORS настройки
- SQL injection защита (через ORM)
- XSS защита

## Полезные команды

### Работа с БД
```bash
# Создать миграцию
alembic revision --autogenerate -m "описание"

# Применить миграции
alembic upgrade head

# Откатить миграцию
alembic downgrade -1
```

### Разработка
```bash
# Запуск dev сервера
uvicorn app.main:app --reload --port 8000

# Запуск тестов
pytest -v

# Форматирование кода
black app/
ruff check app/ --fix
```
