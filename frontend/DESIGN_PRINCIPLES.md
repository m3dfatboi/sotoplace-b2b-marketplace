# Принципы дизайна Sotoplace B2B Marketplace

## Обзор

Этот документ описывает ключевые принципы дизайна интерфейса Sotoplace, основанные на анализе лучших практик крупных маркетплейсов (Ozon, Wildberries, Яндекс Маркет) и фирменном стиле проекта.

---

## 1. Типографика

### Шрифт
- **Семейство**: Inter
- **Варианты**: Regular (400), Medium (500), Semi Bold (600)

### Иерархия размеров

```typescript
// Заголовки
H3: 36px / 600 / 40px line-height
H4: 24px / 600 / 28px line-height
H5: 24px / 500 / 28px line-height

// Подзаголовки
Subtitle3: 16px / 500 / 20px line-height
Subtitle4: 14px / 600 / 18px line-height
Subtitle5: 14px / 500 / 18px line-height

// Основной текст
Body1: 16px / 400 / 20px line-height
Body2: 14px / 400 / 18px line-height

// Вспомогательный текст
Caption2: 12px / 500 / 14px line-height
Caption3: 12px / 400 / 14px line-height
```

### Правила использования
- **H4** - заголовки виджетов и карточек
- **Subtitle5** - основной текст в списках, метки
- **Caption2/3** - вспомогательная информация, метки времени
- **Subtitle4** - акцентированные значения, числа

## 2. Цветовая палитра

### Основные цвета
```css
--color-base-black: #1f1f1f      /* Основной текст */
--color-base-white: #ffffff      /* Фон карточек */
--color-base-furnita: #d18043    /* Акцентный цвет бренда */
```

### Оттенки серого
```css
--color-gray-100: #f5f5f5        /* Фон страницы */
--color-gray-200: #e5e5e5        /* Границы */
--color-gray-300: #d4d4d4        /* Разделители */
--color-gray-400: #a3a3a3        /* Вспомогательный текст */
--color-gray-500: #737373        /* Неактивный текст */
--color-gray-600: #525252        /* Темный вспомогательный */
```

### Статусные цвета
```css
/* Успех */
--color-success-50: #f0f8eb
--color-success-600: #52962a
--color-base-success: #67bb34

/* Ошибка */
--color-error-50: #ffeeee
--color-error-600: #b32c2b
--color-base-error: #e03636

/* Процесс */
--color-process-100: #f6efd5
--color-process-600: #a68a26
--color-base-process: #d0ad2f

/* Furnita */
--color-furnita-600: #a76636
```

### Правила использования цветов
- Основной текст всегда `#1f1f1f`
- Вспомогательный текст `#a3a3a3` или `#737373`
- Акцентный цвет `#d18043` для важных элементов
- Статусные цвета только для индикаторов состояния

## 3. Spacing & Layout

### Базовая сетка: 4px

### Отступы
```
4px   - минимальный gap между мелкими элементами
6px   - gap в inline элементах
8px   - gap между иконками и текстом
10px  - gap в списках
16px  - gap между секциями внутри карточки
20px  - padding карточек, gap между виджетами
```

### Размеры элементов
```
Кнопки:
- Высота: 30px
- Padding: 14px horizontal
- Border radius: 8px

Карточки/Виджеты:
- Border radius: 20px
- Border: 1px solid #e5e5e5
- Padding: 20px
- Фиксированная высота: 236px (для виджетов дашборда)

Иконки:
- Small: 12px
- Medium: 14px
- Large: 20px

Аватары:
- Small: 20px (border-radius: 6px)
- Medium: 30px (border-radius: 8px)
```

## 5. Принципы UX из крупных маркетплейсов

### 5.1 Информационная плотность (Ozon, WB)
- **Виджеты дашборда** должны показывать ключевые метрики с первого взгляда
- Числа крупно, контекст мелко
- Тренды показываются визуально (стрелки, проценты, графики)

### 5.2 Визуальная иерархия (Яндекс Маркет)
- Самое важное - крупнее и контрастнее
- Вспомогательная информация - серым цветом
- Группировка связанных элементов через spacing

### 5.3 Статусы и индикаторы
- Цветовое кодирование состояний (красный/желтый/зеленый)
- Иконки для быстрого распознавания
- Бейджи для количественных показателей

### 5.4 Быстрые действия
- Кнопки действий всегда на виду
- Первичное действие выделено цветом
- Вторичные действия - outline стиль

### 5.5 Адаптивность
- Сетка на 4 колонки для виджетов
- Минимальная ширина виджета: 260px
- Responsive breakpoints:
  - Desktop: 1920px (4 колонки)
  - Laptop: 1440px (3 колонки)
  - Tablet: 1024px (2 колонки)
  - Mobile: 768px (1 колонка)

## 6. Виджеты дашборда

### Обязательные элементы каждого виджета:
1. **Заголовок** (H4, 24px, 600)
2. **Контекст** (Caption2, 12px, серый) - период, статус
3. **Основное значение** - крупно, контрастно
4. **Тренд** - процент изменения с иконкой
5. **Визуализация** - график, диаграмма, список

### Типы виджетов:

#### Метрики (Active Orders, Sales)
- Круговая диаграмма или график
- Крупное число в центре
- Легенда со статусами
- Тренд в процентах

#### Списки (Reminders, Leaders)
- Чекбоксы или ранжирование
- Аватары пользователей
- Разделители между элементами
- Кнопки действий внизу

#### Индикаторы (Load)
- Горизонтальные прогресс-бары
- Цветовое кодирование загрузки
- Метки для каждого индикатора

## 7. Таблицы

### Структура
- Заголовок с фильтрами
- Строки с hover эффектом
- Expandable rows для деталей
- Sticky header при скролле

### Стилизация
```tsx
Header:
- Background: #f5f5f5
- Text: 12px, 500, #737373
- Padding: 12px 16px

Row:
- Border-bottom: 1px solid #e5e5e5
- Padding: 16px
- Hover: background #f5f5f5

Cell:
- Text: 14px, 400, #1f1f1f
- Alignment: left (text), right (numbers)
```

## 8. Правила разработки

### DO ✅
- Использовать точные значения из дизайн-системы
- Следовать типографической иерархии
- Применять цвета по назначению
- Тестировать на разных разрешениях
- Использовать CSS переменные для цветов

### DON'T ❌
- Добавлять тени без указания в дизайне
- Использовать произвольные размеры шрифтов
- Смешивать разные стили кнопок
- Игнорировать spacing систему
- Хардкодить цвета напрямую

## 9. Accessibility

### Контрастность
- Основной текст: минимум 4.5:1
- Крупный текст (18px+): минимум 3:1
- Интерактивные элементы: минимум 3:1

### Интерактивность
- Минимальный размер кликабельной области: 40x40px
- Видимый focus state для клавиатурной навигации
- Aria-labels для иконок без текста

## 10. Performance

### Оптимизация
- Lazy loading для виджетов ниже fold
- Виртуализация для длинных списков
- Debounce для поиска и фильтров
- Кэширование данных дашборда (5 минут)
