# 🚀 Как запустить и протестировать

## Шаг 1: Проверь Backend

Backend должен быть запущен. Проверь:

```bash
docker-compose ps
```

Если не запущен, запусти:

```bash
docker-compose up -d
```

Проверь что API работает:
```bash
curl http://localhost:8000/api/v1/health
```

Должен вернуть: `{"status":"healthy"}`

---

## Шаг 2: Запусти Frontend

Открой **новый терминал** и выполни:

```bash
cd frontend
npm run dev
```

Увидишь:
```
▲ Next.js 16.1.6
- Local:        http://localhost:3000
- Ready in 2.3s
```

---

## Шаг 3: Открой в браузере

### Главная страница (Dashboard)
**http://localhost:3000**

Что увидишь:
- 4 KPI карточки (заказы, активные, выручка, товары)
- Таблица последних 5 заказов
- Кнопки "Создать заказ" и "Добавить товар"

### Страница заказов
**http://localhost:3000/orders**

Что увидишь:
- Поиск по заказам
- Фильтр по статусу
- 3 карточки статистики
- Полная таблица всех заказов

### Каталог товаров
**http://localhost:3000/products**

Что увидишь:
- Поиск по товарам
- Фильтр (все/опубликованные/черновики)
- 3 карточки статистики
- Grid с карточками товаров

---

## Шаг 4: Протестируй функционал

### На странице заказов:
1. ✅ Введи что-то в поиск - таблица обновится
2. ✅ Выбери статус в фильтре - загрузятся заказы с этим статусом
3. ✅ Посмотри как обновляется статистика

### На странице товаров:
1. ✅ Введи что-то в поиск - карточки обновятся
2. ✅ Выбери "Опубликованные" - покажутся только опубликованные
3. ✅ Посмотри статистику

### Навигация:
1. ✅ Кликай по ссылкам в header (Главная, Заказы, Каталог)
2. ✅ Проверь что активная страница подсвечена синим

---

## Что если не работает?

### Backend не отвечает
```bash
docker-compose up -d
docker-compose logs backend
```

### Frontend не запускается
```bash
cd frontend
npm install
npm run dev
```

### Нет данных в таблицах
Нужно засидить тестовые данные:
```bash
docker-compose exec backend python -m scripts.seed_data
```
