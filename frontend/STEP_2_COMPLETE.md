# Frontend - Orders Page (Step 2) ✅

## Что создано

### Страница заказов
**Полный список заказов с фильтрацией и поиском**

#### Компоненты:
1. **Фильтры**:
   - Поиск по номеру заказа, компании
   - Фильтр по статусу (9 статусов)
   - Кнопка создания заказа

2. **Статистика** (3 карточки):
   - Всего заказов
   - Найдено (с учетом фильтров)
   - Общая сумма

3. **Таблица заказов**:
   - Номер заказа
   - Покупатель
   - Поставщик
   - Статус (цветные badges)
   - Сумма
   - Дата создания
   - Дата обновления

#### Функционал:
- ✅ Реальные данные из backend API
- ✅ Поиск в реальном времени (клиентская фильтрация)
- ✅ Фильтрация по статусу (серверная)
- ✅ Автоматический пересчет статистики
- ✅ Hover эффекты на строках
- ✅ Responsive дизайн

---

## Интеграция с Backend

**API endpoint:**
```typescript
GET /api/v1/orders?status={status}&size=50
```

**Параметры:**
- `status` - фильтр по статусу (опционально)
- `size` - количество заказов (50)

## Как тестировать

```bash
cd frontend
npm run dev
```

Откройте: http://localhost:3000/orders

**Проверьте:**
1. Поиск работает
2. Фильтр по статусу работает
3. Статистика обновляется
4. Таблица отображается корректно
